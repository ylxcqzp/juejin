# 数据库设计文档

## 1. 设计规范

### 1.1 通用字段规范

所有业务表必须包含以下通用字段：

| 字段名 | 类型 | 默认值 | 说明 |
|--------|------|--------|------|
| id | BIGINT | 自增 | 主键ID |
| create_time | DATETIME | CURRENT_TIMESTAMP | 创建时间 |
| creator | BIGINT | NULL | 创建人ID |
| update_time | DATETIME | CURRENT_TIMESTAMP ON UPDATE CURRENT_TIMESTAMP | 更新时间 |
| updater | BIGINT | NULL | 更新人ID |
| deleted | TINYINT | 0 | 逻辑删除标识（0-未删除 1-已删除） |

### 1.2 高并发表附加字段

涉及高并发操作的表（如点赞、关注）需增加：

| 字段名 | 类型 | 默认值 | 说明 |
|--------|------|--------|------|
| version | INT | 0 | 乐观锁版本号 |

### 1.3 命名规范

- 表名：小写字母，单词间用下划线分隔，如 `user`、`article`
- 字段名：小写字母，单词间用下划线分隔，如 `user_id`、`create_time`
- 索引名：`idx_字段名` 或 `idx_字段名1_字段名2`
- 唯一索引：`uk_字段名`
- 外键名：`fk_表名_字段名`

### 1.4 数据库配置

- 字符集：`utf8mb4`
- 排序规则：`utf8mb4_unicode_ci`
- 存储引擎：`InnoDB`

### 2.1 用户表 (user)

存储用户基本信息。

| 字段名 | 类型 | 是否为空 | 默认值 | 说明 |

| nickname | VARCHAR(50) | NOT NULL | - | 昵称 |
| avatar | VARCHAR(500) | NULL | - | 头像URL |
| bio | VARCHAR(500) | NULL | - | 个人简介 |
| background_image | VARCHAR(500) | NULL | - | 个人主页背景图 |
| level | TINYINT | NOT NULL | 1 | 用户等级（1-6） |
| points | INT | NOT NULL | 0 | 掘力值 |
| following_count | INT | NOT NULL | 0 | 关注数 |
| follower_count | INT | NOT NULL | 0 | 粉丝数 |
| article_count | INT | NOT NULL | 0 | 文章数 |
| like_count | INT | NOT NULL | 0 | 获赞数 |
| status | TINYINT | NOT NULL | 0 | 状态（0-正常 1-封禁 2-注销中 3-已注销） |

| last_login_time | DATETIME | NULL | - | 最后登录时间 |

| login_fail_count | INT | NOT NULL | 0 | 连续登录失败次数 |

| creator | BIGINT | NULL | - | 创建人ID |
| update_time | DATETIME | NOT NULL | CURRENT_TIMESTAMP | 更新时间 |
| updater | BIGINT | NULL | - | 更新人ID |
| deleted | TINYINT | NOT NULL | 0 | 逻辑删除标识 |

**索引：**
```sql
PRIMARY KEY (id)
UNIQUE KEY uk_email (email)
UNIQUE KEY uk_phone (phone)
KEY idx_nickname (nickname)
KEY idx_status (status)
KEY idx_create_time (create_time)
KEY idx_points (points)
```

## 15. 注意事项

### 15.1 高并发优化

1. **点赞表、关注表**：使用乐观锁（version字段）防止并发问题
2. **计数器字段**：使用Redis缓存，定时同步到MySQL
3. **热点数据**：使用多级缓存（Caffeine + Redis）

### 15.2 索引优化

1. 避免在低区分度字段上创建索引
2. 组合索引遵循最左前缀原则
3. 定期分析慢查询，优化索引

### 15.3 数据安全

1. 敏感字段（密码）使用BCrypt加密
2. 手机号、邮箱脱敏展示
3. 逻辑删除保留数据，定期物理删除

### 15.4 性能监控

1. 监控慢查询（>1秒）
2. 监控表数据量增长
3. 定期优化表结构
