## 3. 微服务拆分与职责

**初期采用5个核心服务，降低复杂度：**

| 服务名 | 职责描述 | 主要存储依赖 |
|--------|----------|--------------|
| gateway-service | 统一入口，路由转发，鉴权校验，跨域处理，限流 | Nacos |
| user-service | 用户认证、用户信息、关注粉丝、成长体系 | MySQL, Redis |
| content-service | 文章管理、标签体系、专栏、搜索、审核流程 | MySQL, ES, Redis |
| social-service | 点赞、收藏、评论、沸点、Feed流、消息通知 | MySQL, Redis |
| operation-service | 签到任务、热榜计算、文件资源、定时任务调度 | MySQL, Redis, OSS |

**后期可拆分：**
- content-service -> content-service + search-service
- social-service -> social-service + message-service + feed-service
- operation-service -> operation-service + resource-service + job-service

### 4.2 Spring Security使用规范

**认证流程：**
1. 用户登录 -> 验证用户名密码 -> 生成JWT Token
2. 请求携带Token -> Gateway解析Token -> 转发到服务
3. 服务从Header获取userId -> 查询用户权限 -> 执行业务

**授权方式：**
- 方法级授权：使用@PreAuthorize注解
- URL级授权：在SecurityConfig中配置
- 数据级授权：在Service层进行权限校验

**示例：**
```java
@PreAuthorize("hasRole('ADMIN')")
@Operation(summary = "删除用户")
@DeleteMapping("/{id}")
public Result<Void> deleteUser(@PathVariable Long id) { }
```

### 5.4 Service层示例

```java
// service/impl/UserServiceImpl.java
@Service
@RequiredArgsConstructor
@Slf4j
public class UserServiceImpl extends ServiceImpl<UserMapper, User> implements UserService {
    
    private final UserMapper userMapper;
    private final RedisTemplate<String, Object> redisTemplate;
    
    @Override
    @Transactional(rollbackFor = Exception.class)
    public UserVO register(RegisterDTO dto) {
        // 检查邮箱是否已注册
        if (lambdaQuery().eq(User::getEmail, dto.getEmail()).exists()) {
            throw new BusinessException(ErrorCode.EMAIL_ALREADY_EXISTS);
        }
        
        // 创建用户 - 使用Hutool的BeanUtil
        User user = BeanUtil.copyProperties(dto, User.class);
        user.setPassword(BCrypt.hashpw(dto.getPassword()));
        user.setLevel(1);
        user.setPoints(0);
        
        save(user);
        log.info("User registered: {}", user.getId());
        
        return BeanUtil.copyProperties(user, UserVO.class);
    }
    
}
```

### 5.5 Controller层示例

```java
// controller/UserController.java
@Tag(name = "用户服务", description = "用户相关接口")
@RestController
@RequestMapping("/api/v1/users")
@RequiredArgsConstructor
public class UserController {
    
    private final UserService userService;
    
    @Operation(summary = "用户注册")
    @PostMapping("/register")
    public Result<UserVO> register(@RequestBody @Validated RegisterDTO dto) {
        UserVO userVO = userService.register(dto);
        return Result.success(userVO);
    }
    
    @Operation(summary = "用户登录")
    @PostMapping("/login")
    public Result<LoginVO> login(@RequestBody @Validated LoginDTO dto) {
        LoginVO loginVO = userService.login(dto);
        return Result.success(loginVO);
    }
    
    @Operation(summary = "获取当前用户信息")
    @GetMapping("/current")
    public Result<UserVO> getCurrentUser(@RequestHeader("X-User-Id") Long userId) {
        UserVO userVO = userService.getUserById(userId);
        return Result.success(userVO);
    }
    
}
```

## 7. 前端架构设计

### 7.1 C端UI方案选择

**推荐方案：Headless UI + Tailwind CSS**

| 对比项 | Element Plus | Headless UI + Tailwind |
|--------|--------------|------------------------|
| 样式定制 | 需覆盖CSS变量，难度大 | 完全自定义，灵活度高 |
| 包体积 | 较大（全量引入） | 按需引入，体积小 |
| 设计一致性 | 组件风格固定 | 完全可控，品牌感强 |
| 开发效率 | 高（组件丰富） | 中（需自行封装） |
| 适用场景 | 管理后台 | C端用户界面 |

**结论：** C端页面追求独特设计和用户体验，建议使用 **Headless UI + Tailwind CSS**，管理后台使用 Element Plus。

### 7.2 前端技术栈


## 9. 安全策略

### 9.1 认证安全

- **密码加密**：BCrypt，强度10
- **Token策略**：
  - Access Token：有效期30分钟
  - Refresh Token：有效期7天，只能使用一次
- **登录安全**：
  - 连续5次失败锁定15分钟
  - 异地登录提醒
  - 单设备登录（可选）

### 9.2 接口安全

- **防重放攻击**：请求头携带时间戳，有效期5分钟
- **限流策略**：
  - 登录接口：5次/分钟
  - 普通接口：100次/分钟
  - 搜索接口：30次/分钟
- **敏感操作二次验证**：修改密码、绑定手机需要验证码

### 9.3 数据安全

- **传输加密**：全站HTTPS，TLS 1.3
- **敏感数据脱敏**：
  - 手机号：138****8888

### 11.2 建议补充的非业务功能

#### 11.2.1 高优先级

##### （1）Sentinel 接口限流

**现状**：仅引入依赖，未配置具体规则

**建议实现**：
```java
// 网关层限流规则配置
@Configuration
public class SentinelConfig {

    @PostConstruct
    public void init() {
        // 登录接口：5次/分钟
        FlowRuleManager.loadRules(Collections.singletonList(
            createFlowRule("login", 5, 60)
        ));
        // 普通接口：100次/分钟
        FlowRuleManager.loadRules(Collections.singletonList(
            createFlowRule("api", 100, 60)
        ));
        // 搜索接口：30次/分钟
        FlowRuleManager.loadRules(Collections.singletonList(
            createFlowRule("search", 30, 60)
        ));
    }

    private FlowRule createFlowRule(String resource, int count, int interval) {
        FlowRule rule = new FlowRule(resource);
        rule.setCount(count);
        rule.setIntervalSec(interval);
        rule.setControlBehavior(RuleConstant.CONTROL_BEHAVIOR_DEFAULT);
        return rule;
    }
}
```

**或使用 Sentinel Dashboard 配置流控规则**。

##### （2）分布式事务（Seata）

**适用场景**：跨服务的数据一致性，如点赞文章时同时更新文章点赞数

**建议引入**：
```xml
<!-- 添加依赖 -->
<dependency>
    <groupId>com.alibaba.cloud</groupId>
    <artifactId>spring-cloud-starter-alibaba-seata</artifactId>
</dependency>
```

**配置 Seata Server 并在需要事务的方法上添加 `@GlobalTransactional` 注解**。

##### （3）接口幂等性

**适用场景**：防止重复提交，如重复点赞、重复支付

**建议实现方案**：
- 基于 Redis Token：请求前获取 Token，请求时验证并删除
- 基于数据库唯一索引：如用户对同一文章只能点赞一次
- 基于分布式锁：同接口并发控制

```java
// 示例：基于 Redis 的幂等性注解
@Target(ElementType.METHOD)
@Retention(RetentionPolicy.RUNTIME)
public @interface Idempotent {
    long expireSeconds() default 60;
}
```

##### （4）请求日志追踪

**适用场景**：全链路日志追踪，问题定位

**建议实现**：
- MDC + TraceId：每个请求生成唯一 TraceId
- 或接入 SkyWalking（已在技术选型中规划）

```java
// Filter 中生成 TraceId
String traceId = UUID.randomUUID().toString();
MDC.put("traceId", traceId);
```
